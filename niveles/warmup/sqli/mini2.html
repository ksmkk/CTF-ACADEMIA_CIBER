SQL Injection - Mini Desafio 2

Por Tiempo (Time-Based)

Objetivo

Descubre cómo confirmar una SQL Injection midiendo el tiempo de respuesta del servidor. Este es un ataque más silencioso que no depende de mensajes de error.

Informacion: SQLi Time-Based

Blind SQL Injection (por Tiempo) ocurre cuando:

  • El servidor no muestra mensajes de error
  • Inyectas código que hace que el servidor espere (comando SLEEP o similar)
  • Si espera más tiempo = tu inyección funcionó
  • Si responde rápido = no funcionó

Ejemplo: Si inyectas '; SLEEP(5) -- - y la pagina tarda 5 segundos en responder, confirmaste que hay SQLi, aunque no veas mensajes de error!

Comparativa

❌ Consulta Normal (Fast)

El servidor responde rápido:

Intenta cualquier contraseña
Tiempo: 0ms

✅ Con SLEEP (Time-Based SQLi)

Inyecta SLEEP para detectar vulnerabilidad:

Intenta: ' SLEEP(3) -- -
Tiempo: 0ms

Como Funciona Time-Based SQLi

-- Consulta normal:
SELECT * FROM users WHERE id = 1
-- Consulta inyectada (Time-Based):
SELECT * FROM users WHERE id = 1; SLEEP(5) -- -

-- Resultado: El servidor espera 5 segundos antes de responder

Ventaja: No necesitas ver un mensaje de error para confirmar SQLi. Desventaja: Es más lento (necesitas esperar).
