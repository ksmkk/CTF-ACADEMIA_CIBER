XSS (Cross-Site Scripting) - Mini Desafio 1

Reflected XSS

Objetivo

Descubre cómo código malicioso puede ejecutarse en el navegador del usuario. Observa cómo un campo de búsqueda sin validación puede ser peligroso.

Informacion: XSS (Cross-Site Scripting)

XSS ocurre cuando un atacante inyecta código JavaScript en una página web. El navegador ejecuta ese código como si fuera parte de la página legítima.

  • El atacante inyecta: <script>alert('Hackeado!')</script>
  • El servidor renderiza el HTML sin validar
  • El navegador ejecuta el script automáticamente
  • El usuario ve una alerta (o peor, pierde sus datos)

Peligro: En un ataque XSS real, el atacante puede robar cookies, sesiones, contraseñas, o realizar acciones en nombre del usuario.
