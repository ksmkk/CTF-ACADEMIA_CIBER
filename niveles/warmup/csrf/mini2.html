CSRF Attack - Mini Desafio 2

Sesion Activa

Objetivo

Descubre cómo un sitio malicioso puede usar tu sesión activa para ejecutar acciones sin tu permiso. Verás cómo protegerse contra esto.

Informacion Basica: Proteccion CSRF

Para prevenir CSRF, los formularios modernos usan un mecanismo de seguridad llamado CSRF Token (o Token CSRF).

  • El servidor genera un token único para cada sesión
  • El servidor incluye este token en cada formulario
  • Cuando envías el formulario, debes incluir el token
  • Si no hay token o es inválido, la acción se rechaza

Por que funciona? Un sitio malicioso no puede robar el token porque esta protegido por políticas del navegador (Same-Origin Policy).

Comparativa: Con y Sin Proteccion

Sin CSRF Token

Cualquier sitio puede enviar este formulario:

<form action="/transferir">
  <input name="monto" value="1000">
  <input name="destino" value="hacker">
  <button>Enviar</button>
</form>

Un sitio atacante puede copiar este HTML y ejecutarlo con tu sesion activa.

Con CSRF Token

El servidor añade un token único:

<form action="/transferir">
  <input name="monto" value="1000">
  <input name="destino" value="hacker">
  <input name="token"
    value="a1b2c3d4e5f6">
  <button>Enviar</button>
</form>

Un sitio atacante NO puede obtener el token (Same-Origin Policy), por lo que no puede enviar el formulario valido.
